Guía práctica – Clase 23

Actividad:
Integración Frontend-Backend y consumo de la API propia.

Repositorio:
Repositorio dispuesto por el docente.

Importante:
Seguimos trabajando en el proyecto único de Semana 5.

No debes trabajar directamente en main.

Cada estudiante debe trabajar en su rama personal con el formato:

nombres_apellidos

Evidencia:
La evidencia de esta actividad es únicamente el push de tu rama personal a GitHub.

Pasos obligatorios:

1. Entrar al proyecto

cd programadores-para-la-paz-mini-sitio-comunitario

2. Actualizar main local

git checkout main
git pull --ff-only origin main

3. Volver a la rama personal

git checkout nombres_apellidos

4. Incorporar los cambios de main a la rama personal

git merge main

5. Verificar la rama actual

git branch

6. Instalar dependencias si hace falta

npm install

7. Levantar el mini-sitio

npm run sitio

8. Abrir en el navegador

http://localhost:3000

9. Probar rutas directas:

http://localhost:3000/api/mensajes
http://localhost:3000/api/calendario
http://localhost:3000/api/resumen

10. Probar botones:

Cargar mensajes
Cargar calendario
Cargar resumen

11. Revisar server.js

Identificar:
GET /api/mensajes
GET /api/calendario
GET /api/resumen

12. Revisar public/app.js

Identificar:
fetch
async
await
respuesta.json()
for
innerHTML
appendChild

13. Modificar un mensaje en data/mensajes.json

14. Reiniciar servidor con Ctrl + C y luego:

npm run sitio

15. Verificar que el cambio aparece en el tablero

16. Completar archivos:

semana5/pruebas-integracion-frontend-backend.txt
semana5/mapa-flujo-frontend-backend.txt
semana5/revision-mensajes-dinamicos.txt
semana5/checklist-api-propia.txt
semana5/reflexion-integracion-fe-be.txt
semana5/bitacora-git-proyecto-vivo.txt

17. Revisar cambios

git status

18. Agregar cambios

git add .

19. Crear commit

git commit -m "Clase 23 integracion frontend backend API propia"

20. Publicar rama

git push
